## 1. 기획 의도 및 배경

**"왜 약속을 잡을 때마다 지도 앱, 맛집 검색, 단톡방을 오가야 할까요?"**

이 프로젝트는 별도의 앱 서비스가 아닙니다. 국민 메신저 카카오톡에 탑재될 '약속 잡기 기능'의 프로토타입입니다.

사용자가 대화 도중 앱을 이탈하지 않고, 채팅방 내에서 즉시 약속 장소를 정할 수 있는 Seamless한 경험을 구현하는 데 초점을 맞추었습니다.

### 👥 기존 카카오톡 경험의 확장
<img width="1024" height="559" alt="2" src="https://github.com/user-attachments/assets/b47954ec-5341-4c5c-84df-fe57cae7cf13" />


- **AS-IS**: "강남역 어때?" → (지도 앱 실행) → (경로 검색) → (캡처 후 공유) → "여기 너무 먼데?" → (무한 반복)
- **TO-BE**: 채팅방 `+` 버튼 클릭 → **[모여라]** 실행 → 각자 위치 입력 → **AI가 공평한 중간 지점 추천** → 약속 확정

## 2. 핵심 가치

### 🔗 카카오톡 내부에서 맥락의 연결

- 외부 링크를 타고 웹브라우저로 나가는 것이 아니라, **카카오톡 인앱** 환경에서 네이티브 기능처럼 동작합니다.
- 단톡방 멤버가 곧 약속 참여자가 되므로, 별도의 친구 초대나 가입 절차가 필요 없습니다.

### ⏱️ 이동 시간의 공평함

- 단순히 지리적인 중간(거리 중심)이 아닌, 대중교통을 이용한 **실제 이동 시간**을 기준으로 장소를 추천합니다.
- 모두가 납득할 수 있는 '시간적 합의점'을 도출하여 장소 선정의 갈등을 줄입니다.

### 🧠 AI 어시스턴트를 활용한 필터링 적용

- "나 오늘 차 가져가", "비 오니까 걷기 싫어" 같은 자연어 대화를 분석하여, **주차 가능 여부**나 **역세권** 필터를 자동으로 적용합니다.

## 4. 기술적 구현

- **Frontend**: HTML5/CSS3/Vanilla JS (별도 빌드 과정 없이 즉시 배포/수정 용이)
- **Backend (BaaS)**: Supabase (참여자 상태 실시간 동기화 및 세션 관리)
- **AI Engine**: Google Gemini API (사용자 상황 맥락 분석 및 조건 필터링)
- **Map/Location**: Kakao Map SDK, Local API (장소 검색 및 좌표 변환)

## 5. 향후 확장 계획

이 기능은 단순한 장소 추천을 넘어, **카카오 생태계의 다양한 서비스들을 '약속'이라는 맥락으로 묶어내는 허브(Hub)** 역할을 수행합니다.
<img width="1024" height="559" alt="7" src="https://github.com/user-attachments/assets/328ded27-5d37-440f-aff6-15ff4b54dedb" />

### Phase 1: 이동과 장소의 연결 (Mobility & Map)

- **Kakao T 연동**: 추천된 장소 결과 화면에서 **[카카오 T 택시 호출]** 버튼을 노출합니다. 목적지가 자동 입력되어 원클릭으로 이동 수단을 확보합니다.
- **Kakao Map 길찾기**: "가장 빠른 경로"를 바로 카카오맵으로 연결하여 내비게이션 기능을 수행합니다.

### Phase 2: 일정과 금융의 결합 (Calendar & Pay)

- **톡캘린더 자동 등록**: 확정된 약속 시간과 장소를 참여자 전원의 **톡캘린더**에 자동으로 등록하고 알림을 설정합니다.
- **예약금/정산 (Kakao Pay)**: 식당 예약이 필요한 경우, **카카오페이**를 통해 예약금을 결제하거나 1/N 정산하기 기능을 즉시 실행합니다.

### Phase 3: 재미와 관계 강화 (Gift & Fun)

- **지각비 베팅 기능**: 약속 장소 도착 시간을 체크하여, 가장 늦게 도착한 친구가 벌칙을 수행하는 게이미피케이션 요소를 도입합니다.
- **카카오톡 선물하기 (Kakao Gift)**: 가장 멀리서 오는 친구에게 "고생했어" 의미로 커피 쿠폰을 보낼 수 있는 **선물하기 큐레이션**을 제공합니다.
